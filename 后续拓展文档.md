## 1. 收藏与书签

### 需求
用户可以对文档添加收藏标记，在独立视图中查看收藏列表。

### 数据库变更
```sql
ALTER TABLE documents ADD COLUMN is_favorite BOOLEAN DEFAULT FALSE;
CREATE INDEX idx_documents_favorite ON documents(user_id, is_favorite) WHERE is_favorite = TRUE;
```

### 涉及文件
- `backend/models/document.py` — 添加 `is_favorite` 字段
- `backend/services/document_service.py` — 添加 `toggle_favorite()`、`list_favorites()`

- `frontend/src/views/Documents.vue` — 收藏按钮、收藏筛选
- `frontend/src/views/Favorites.vue` — 新建收藏列表页

### 实现要点
- 使用布尔字段而非关联表，简化查询
- 前端用星标图标切换，乐观更新 UI
- 列表页增加"仅收藏"筛选选项

## 2. 分类与标签系统

### 需求
支持多级分类文件夹 + 多标签，便于文档组织。

### 数据库变更
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    name VARCHAR(100) NOT NULL,
    parent_id INTEGER REFERENCES categories(id),
    icon VARCHAR(50),
    sort_order INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_categories_user ON categories(user_id);
CREATE INDEX idx_categories_parent ON categories(parent_id);

CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    name VARCHAR(50) NOT NULL,
    color VARCHAR(7),  -- hex color
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, name)
);

CREATE TABLE document_tags (
    document_id INTEGER NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    tag_id INTEGER NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (document_id, tag_id)
);

ALTER TABLE documents ADD COLUMN category_id INTEGER REFERENCES categories(id);
CREATE INDEX idx_documents_category ON documents(category_id);
```

### 涉及文件
- `backend/models/category.py`、`tag.py` — 新模型
- `backend/models/document.py` — 添加 `category_id` 外键
- `backend/routers/categories.py`、`tags.py` — CRUD 端点
- `backend/services/category_service.py`、`tag_service.py` — 业务逻辑
- `frontend/src/components/CategoryTree.vue` — 树形分类组件
- `frontend/src/components/TagInput.vue` — 标签输入组件

### 实现要点
- 分类支持嵌套（parent_id 自引用），前端用树形组件展示
- 标签按用户隔离，名称唯一约束
- 文档可属于一个分类、多个标签
- 搜索时可按分类/标签筛选

## 3. 首页仪表盘

### 需求
首页展示：最近访问文档、最常访问文档、文档统计、快速操作。

### 数据库变更
```sql
ALTER TABLE documents ADD COLUMN view_count INTEGER DEFAULT 0;
ALTER TABLE documents ADD COLUMN last_viewed_at DATETIME;

CREATE TABLE activity_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    document_id INTEGER REFERENCES documents(id) ON DELETE SET NULL,
    action VARCHAR(50) NOT NULL,  -- view, upload, delete, search
    metadata TEXT,  -- JSON

```

### 涉及文件
- `backend/models/activity.py` — 活动日志模型
- `backend/routers/dashboard.py` — 仪表盘 API

- 文档查看时递增 `view_count`，更新 `last_viewed_at`
- 统计数据可缓存（Redis 或内存 TTL 缓存）
- 活动日志异步写入，避免阻塞主请求

### API 设计
```

GET /api/v1/dashboard/top      → 访问最多的 10 个文档
GET /api/v1/dashboard/activity → 最近活动记录
```

## 4. 文档版本管理

### 需求
文档重新上传时保留历史版本，支持回滚。

### 数据库变更
```sql
CREATE TABLE document_versions (
```

### 涉及文件
- `backend/models/document_version.py`
- `backend/services/version_service.py` — 创建版本、回滚、列表
- `backend/routers/documents.py` — 添加版本相关端点

### 实现要点
- 每次重新处理/重新上传前，将当前版本存入 `document_versions`
- 回滚时复制版本文件到主文档路径，更新元数据
- 限制最大版本数（如 10），超出时删除最旧版本及其文件

## 7. Milvus 向量搜索集成

### 需求
将文档文本向量化存入 Milvus，支持语义相似搜索。

### 架构设计
```
文档上传 → 文本提取 → 分块(chunk) → Embedding API → Milvus 存储
                                                      ↓
用户搜索 → 查询向量化 → Milvus 相似搜索 → 返回相关文档/段落
```

### 新增依赖
- `pymilvus` — Milvus Python SDK
- Embedding 服务（OpenAI / 本地模型 / 自部署）

### 涉及文件
- `backend/services/embedding_service.py` — 文本分块 + 向量化
- `backend/services/vector_service.py` — Milvus CRUD
- `backend/routers/vector_search.py` — 语义搜索端点
- `backend/config.py` — Milvus 连接配置
- `frontend/src/components/SemanticSearch.vue`

### 数据库变更
```sql
-- Milvus collection schema
-- fields: id (int64), doc_id (int64), chunk_index (int32), embedding (float_vector[1536]), text (varchar)
```

### 实现要点
- 文本分块策略：按段落或固定长度（512 tokens），重叠 50 tokens
- Embedding 异步生成，处理完成后更新 Milvus
- 搜索返回 top-k 结果，附带原文片段
- 删除文档时同步清理 Milvus 数据
- Milvus 连接失败时降级到 FTS5 搜索

### 环境变量
```
MILVUS_HOST=localhost
MILVUS_PORT=19530
MILVUS_COLLECTION=documents
EMBEDDING_API_URL=https://api.openai.com/v1/embeddings
EMBEDDING_API_KEY=sk-xxx
EMBEDDING_MODEL=text-embedding-3-small
```

## 8. LLM 智能摘要与问答

### 需求
- 自动为文档生成摘要
- 基于文档内容的问答（RAG）

### 架构设计
```
文档处理完成 → 提取文本 → LLM 生成摘要 → 存入 document.summary

用户提问 → 语义搜索相关段落 → 构建 prompt → LLM 回答 → 返回答案 + 引用来源
```

### 涉及文件
- `backend/services/llm_service.py` — LLM API 调用封装
- `backend/services/summary_service.py` — 摘要生成
- `backend/services/qa_service.py` — RAG 问答
- `backend/routers/chat.py` — 问答端点（已有占位）
- `backend/models/document.py` — 添加 `summary` 字段
- `frontend/src/views/Chat.vue` — 对话界面

### 数据库变更
```sql
ALTER TABLE documents ADD COLUMN summary TEXT;

CREATE TABLE chat_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    document_id INTEGER REFERENCES documents(id) ON DELETE SET NULL,
    role VARCHAR(20) NOT NULL,  -- user, assistant
    content TEXT NOT NULL,
    sources TEXT,  -- JSON array of referenced chunks
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_chat_user ON chat_history(user_id, created_at DESC);
```

### 实现要点
- 摘要生成在文档处理完成后异步执行
- RAG 流程：搜索 → 构建 context → 调用 LLM → 解析回答
- 支持流式输出（SSE）
- 对话历史持久化，支持上下文连续对话
- LLM 调用限流，避免滥用

### 环境变量
```
LLM_API_URL=https://api.openai.com/v1/chat/completions
LLM_API_KEY=sk-xxx
LLM_MODEL=gpt-4o-mini
LLM_MAX_TOKENS=2048
```


- 超级用户可配置限流豁免

## 数据库迁移策略

后续所有数据库变更使用 Alembic 管理：

```bash
# 安装 Alembic
pip install alembic

# 初始化
alembic init alembic

# 生成迁移
alembic revision --autogenerate -m "add favorites"

# 执行迁移
alembic upgrade head

# 回滚
alembic downgrade -1
```

### 注意事项
- SQLite 不支持 ALTER TABLE ADD CONSTRAINT，需使用表重建方式
- 生产环境迁移前备份数据库
- 测试环境先验证迁移脚本
